# Telegram-бот со статистикой и курсами валют

Простой Telegram-бот на aiogram: здоровается, считает визиты, отдаёт статистику по пользователям и показывает курсы валют в рублях с кэшированием.

## Что умеет

- `/start` — приветствует пользователя и показывает, сколько раз он уже запускал бота (количество визитов).
- `/stats` — показывает счётчик визитов конкретного пользователя и общее количество уникальных пользователей бота.
- `/top` — выводит топ-5 пользователей по количеству вызовов `/start` (если такие пользователи есть).
- `/курс USD` (или другой код валюты, например `EUR`, `GBP`) — показывает курс выбранной валюты в рублях. Данные кэшируются в отдельной базе данных: если сохранённому курсу больше часа, бот асинхронно запрашивает свежее значение из внешнего API ([`open.er-api.com`](https://open.er-api.com/) — открытый эндпоинт ExchangeRate-API, без ключа) и обновляет запись в кэше. Если курс свежий (моложе часа), он отдаётся прямо из базы без обращения к API.
- Любой обычный текст — отвечает тем же сообщением (режим «эхо»).

Под капотом бот ведёт учёт уникальных пользователей и количества их визитов в базе данных SQLite. Курсы валют хранятся в отдельной кэширующей базе в таблице `rate_cache`, где для каждой валюты лежат её последний курс к рублю и время обновления.

## Стек

- Python
- [aiogram](https://docs.aiogram.dev/) 3.28.2
- [aiohttp](https://docs.aiohttp.org/) — асинхронные HTTP-запросы к внешнему API
- [asyncio](https://docs.python.org/3/library/asyncio.html) — асинхронная работа бота и сетевых запросов
- Внешний API — ExchangeRate-API, открытый эндпоинт [`open.er-api.com`](https://open.er-api.com/) (open-access, без ключа и регистрации) для получения курсов валют
- SQLite — модуль `sqlite3` из стандартной библиотеки Python (отдельная установка не требуется). Используется для статистики визитов и для кэширующей таблицы `rate_cache`

## Как запустить

1. Клонируй репозиторий и перейди в папку проекта:

   ```bash
   git clone <ссылка-на-репозиторий>
   cd week_05_bot
   ```

2. Создай и активируй виртуальное окружение:

   ```bash
   python -m venv .venv
   ```

   - Linux / macOS:

     ```bash
     source .venv/bin/activate
     ```

   - Windows:

     ```bash
     .venv\Scripts\activate
     ```

3. Установи зависимости:

   ```bash
   pip install -r requirements.txt
   ```

4. **Настрой токен.** В репозитории нет файла `config.py` (он в `.gitignore`, чтобы токен не попал в открытый доступ). Вместо него лежит шаблон `config.example.py`. Скопируй его в `config.py` и впиши свой токен:

   - Linux / macOS:

     ```bash
     cp config.example.py config.py
     ```

   - Windows:

     ```bash
     copy config.example.py config.py
     ```

   Затем открой `config.py` и вставь токен, который выдаёт [@BotFather](https://t.me/BotFather).

   > Без этого шага бот не запустится — `config.py` обязателен.

5. Запусти бота:

   ```bash
   python main.py
   ```

   > Базы данных SQLite создаются автоматически при первом запуске — никаких дополнительных действий не требуется. База статистики (`visits.db`) и кэш курсов с таблицей `rate_cache` добавлены в `.gitignore`, поэтому в репозиторий не попадают.
